# Phase 3: フロントエンドデザイン全面改修（新規デザインパターン制作）

## 重要事項（必読）

**本プロンプトは、既存のデザイン実装を参照・維持せず、frontend-designスキルのガイドラインに基づいて完全に新しいデザインパターンを制作するものである。**

既存コードへのアンカリング（現状維持バイアス）を避けるため、以下を厳守すること：
- 既存の `src/components/` 内のコードを参照しない
- 既存のスタイル定義を流用しない
- 新規ディレクトリ `src/design-patterns/` に新しいデザインを構築する
- デザイン決定はすべてスキルのガイドラインとサイト目的から導出する

---

## Phase 0: スキル準備（必須・中断条件あり）

### Step 1: frontend-designスキルの発見

以下のコマンドでスキルの存在を確認すること：

```bash
# スキルディレクトリを検索
find /mnt/skills -name "SKILL.md" 2>/dev/null | grep -i frontend

# または
ls -la /mnt/skills/public/
```

### Step 2: スキルの読み込みと内容確認

SKILL.mdファイルを発見したら、**必ず全文を読み込み**、以下の項目を確認・報告すること：

1. **AIスロップ回避リスト** - 避けるべきパターンは何か
2. **推奨されるデザイン方向性（Tone）** - どのような選択肢があるか
3. **タイポグラフィガイドライン** - フォント選定の指針
4. **アニメーション設計の原則** - 推奨されるアプローチ
5. **カラーシステムの構築方法** - 定義すべき要素

### ⚠️ 中断条件

**以下のいずれかに該当する場合、作業を中断し報告すること：**

- [ ] frontend-designスキルのSKILL.mdが見つからない
- [ ] スキルの内容を読み込めない
- [ ] スキルの内容が不完全または破損している
- [ ] 上記5項目のいずれかがスキルに記載されていない

**中断時の報告フォーマット：**
```

```

## Phase 1: サイト情報の理解（デザインの根拠）

### サイトの目的

AI動画制作・デザインサービスの受注獲得を目的としたポートフォリオサイト。
訪問者（潜在顧客）に制作者のスキルと実績を訴求し、問い合わせにつなげることがゴール。

### ターゲット顧客

| 顧客セグメント | 特徴 |
|---------------|------|
| D2Cブランド | 化粧品・美容系、ビジュアル重視 |
| オンラインスクール | 教育系サービス、信頼性重視 |
| スタートアップ | アプリ/SaaS企業、先進性重視 |
| 店舗経営者 | 飲食・小売、コスパ・実績重視 |

### 制作者の強み（訴求ポイント）

1. **キャラクター一貫性技術** - AI動画でキャラクターの一貫性を保てる（市場で希少）
2. **12年のデザイン経験** - 品質担保、部署長レベルの実績
3. **ワンストップ対応** - 企画から動画編集、デザインまで一貫

### 合意済みデザイン方向性

| 要素 | 設定値 | 意図 |
|------|--------|------|
| トーン | **Tech Bold + Creative Edge** | 先進性とクリエイティブさの両立 |
| 基調色 | ダークテーマ（#0A0A0F〜#14141F） | プロフェッショナル感、映像が映える |
| Primary | シアン系（#00D4FF） | AI・テクノロジー・先進性 |
| Accent | オレンジ系（#FF6B35） | クリエイティブ・情熱・行動喚起 |
| 速度感 | ダイナミック寄り | 先進性の表現、期待感の醸成 |
| フォント | モダン・シャープ | 専門性と読みやすさの両立 |

### サイト構成（5セクション）

1. **Hero** - キャッチコピー + 代表動画 + CTA
2. **Services** - 3つのサービスカード（AI動画/AI漫画/デザイン）
3. **Works** - カテゴリ別実績ギャラリー
4. **Profile** - 自己紹介 + 強み + 制作フロー + SNS
5. **Contact** - お問い合わせフォーム

## Phase 2: デザインアーキテクチャ策定

### 指示

frontend-designスキルのガイドラインと上記サイト情報を照合し、以下のデザインアーキテクチャを策定すること。

**策定すべき項目：**

1. **カラーシステム** - 背景、サーフェス、テキスト、Primary、Accent、状態色
2. **タイポグラフィシステム** - フォントファミリー、サイズスケール、ウェイト使い分け
3. **スペーシングシステム** - 余白のスケール定義
4. **アニメーション設計** - 使用するライブラリ、イージング、duration定義
5. **コンポーネント設計方針** - ボタン、カード、セクション区切りの方針

**出力フォーマット：**

```markdown
## デザインアーキテクチャ

### 1. カラーシステム
（CSS変数形式で定義）

### 2. タイポグラフィシステム
（フォント選定理由を含む）

### 3. スペーシングシステム
（8px基準のスケール等）

### 4. アニメーション設計
（具体的なduration、easing値を含む）

### 5. コンポーネント設計方針
（各コンポーネントの視覚的特徴）

### AIスロップ回避チェック
- [ ] 汎用フォント（Inter, Roboto, Arial）を避けている
- [ ] 紫グラデーション×白背景を避けている
- [ ] テンプレート的なレイアウトを避けている
- [ ] ターゲットユーザーを意識した設計になっている
```

## Phase 3: 新規デザインパターン制作

### 作業ディレクトリ

**既存コードとは完全に分離された新規ディレクトリを作成すること：**

```
src/design-patterns/
├── design-system/
│   ├── colors.css          # カラートークン
│   ├── typography.css      # タイポグラフィ
│   ├── spacing.css         # スペーシング
│   └── animations.ts       # アニメーション定義
├── components/
│   ├── Button.tsx
│   ├── Card.tsx
│   ├── SectionHeading.tsx
│   └── Container.tsx
├── sections/
│   ├── HeroPattern.tsx
│   ├── ServicesPattern.tsx
│   ├── WorksPattern.tsx
│   ├── ProfilePattern.tsx
│   └── ContactPattern.tsx
└── PreviewPage.tsx          # 全パターンを確認できるプレビューページ
```

### 実装指示

1. **design-system/** にデザイントークンを定義
2. **components/** に再利用可能なUIコンポーネントを作成
3. **sections/** に各セクションのデザインパターンを実装
4. **PreviewPage.tsx** で全体を通して確認できるようにする

### 重要：既存コードを参照しない

- `src/components/` の既存ファイルを見ない
- `src/app/page.tsx` の現在の実装を参照しない
- globals.css の既存定義を流用しない
- **すべてをスキルのガイドラインから新規に構築する**

## Phase 5: 検証（Verify）

### 検証項目

Playwright MCP または手動で以下を確認：

1. **表示確認**
   - [ ] `/design-preview` にアクセスしてページが表示される
   - [ ] 全5セクションのパターンが確認できる
   - [ ] コンソールエラーがない

2. **デザイン品質確認**
   - [ ] AIスロップパターンに該当していない
   - [ ] Tech Bold + Creative Edgeのトーンが表現されている
   - [ ] アニメーションが意図通り動作する

3. **レスポンシブ確認**
   - [ ] モバイル（375px）での表示
   - [ ] タブレット（768px）での表示
   - [ ] デスクトップ（1280px+）での表示
